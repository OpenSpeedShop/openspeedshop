Open|SpeedShop Quick Start Information

Selecting an Experiment Type

The Summary of Experiments table below shows the possible experiments you can perform using the Open|SpeedShop tools and the reasons why you might want to choose a specific experiment. The Clues column shows when you might use an experiment. The Data Collected column indicates performance data collected by the experiment. For detailed information on the experiments, see the relevant section in the remainder of this chapter.    Open|SpeedShop provides wizards in the Graphical User Interface that "walk" users through the process of selecting the performance data to be gathered.   The wizards ask a set of "plain" language questions to focus on the desired performance monitoring experiment and, therefore also, the desired performance monitoring information to be gathered and analyzed.

High user CPU time.

Actual CPU time at the source line, machine instruction, and function levels by sampling the program counter at 100 samples per second intervals.

Open|SpeedShop Invocation Command ”openss"

The tools that are the basis for Open|SpeedShop are the performance tool graphical user interface (GUI), the interactive command line (CLI), and the batch command.   These are the Open|SpeedShop tools that have user interfaces.  The Open|SpeedShop program will be invoked by the user typing the "openss" command.  When the user invokes Open|SpeedShop command there are four options for its mode of operation:

  • openss -offline

This invocation of Open|SpeedShop uses the offline mode of operation which links our performance data collection modules with your application and collects the performance data you specify.   This is now the default mode of operation so the "-offline" is optional.  See the example below.  One can also use the convenience commands: osspcsamp, ossusertime, osshwc, osshwctime, ossio, ossiot, ossmpi, ossmpit, ossmpiotf, ossfpe.  These commands use a very simple syntax and are meant to be an easier way to invoke the offline functionality.   See the documentation for these convenience commands here.


  • openss -gui

This invocation of Open|SpeedShop causes the GUI to be raised then a command panel is also created.  This command panel window becomes the interactive CLI window.  Under this invocation Open|SpeedShop interactive commands can be entered into the GUI's command panel and have the same effect as if they were entered under the "openss -cli" mode of operation.  This option can gather data using the offline or online instrumentation mechanism.

  • openss -cli:

This invocation of Open|SpeedShop causes the window terminal becomes the interactive CLI window.   This option can gather data using the offline or online instrumentation mechanism.


  • openss -batch:

This invocation of Open|SpeedShop causes Open|SpeedShop to execute a performance experiment specified by additional arguments, directly without user interaction.   The -batch operation can be used in scripts and batch processing environments.

  • openss -online

This invocation of Open|SpeedShop uses the online/dynamic mode of operation which uses dynamic instrumentation to insert the Open|SpeedShop  performance data collection modules into the user application and collects the performance data you specify.   This is not the default mode of operation so the "-online" is must be specified. 


Open|SpeedShop Invocation Example Information


Open|SpeedShop can be invoked in a number of ways.  This section gives a few examples that will hopefully help the first time users to get started and serve as an introduction to the sections in the full User Guide about Open|SpeedShop tool usage.

As stated above in the "openss" command section, the Open|SpeedShop command "openss" will initiate the Open|SpeedShop performance tool.   The type of execution of Open|SpeedShop arguments to the openss command are "-batch", "-cli", “gui”, and "-offline".  The default type of execution is GUI.   "openss" with no parameters will invoke the Open|SpeedShop GUI tool in wizard mode.   The following examples show a few simple typical usage scenarios for Open|SpeedShop.



Key command options

-f “executable path and arguments”

-f executable

Specify the executable/application that performance data will be gathered for/on.


Quotes around the executable are required if arguments or input files are specified.


Note: Specify the executable and arguments just as you normally would to run

the application

experiment_type

The type of performance data that you would like to gather about your program.


See the section above about “Selecting an Experiment Type”.



Example 1:  Using offline

Using the Open|SpeedShop offline mode of operation and assuming you know the executable you want to gather performance data for and the experiment you want to run (User Time)

General syntax:
prompt :> openss [–offline] -f “<pathToExecutable>/executable_name” <experiment_type>

Real example:

prompt :> openss –offline –f “/opt/openmpi-1.4.1/bin/orterun –np 64 ./smg2000 –n 100 100 100” usertime

The above commands will execute as immediate commands on localhost.  The program output will come to the screen followed by the performance data report generated by Open|SpeedShop. An Open|SpeedShop database file containing the results is also created and is available for post-mortem viewing. Look for a file with the “.openss” suffix. All Open|SpeedShop database files have the “.openss” suffix.


Quotes are used around the –f option if the executable has parameters or arguments, such as in the MPI application shown above.


The convenience command equivalent to the above is:
prompt :> ossusertime “/opt/openmpi-1.4.1/bin/orterun –np 64 ./smg2000 –n 100 100 100”

NOTE:  See the convenience function documentation here.
NOTE : See the offline environment variable information.  These environment variable allow the user to control the sampling rate, the functions that are traced, and other user options.

Example 2:  Using GUI

Using the Open|SpeedShop GUI,  you know your executable and the experiment you want to run (MPI)


General syntax:
prompt:> openss -gui [-online | -offline] -f <pathToExecutable>/executable_name <experiment_type>


Real example:

prompt:> openss -gui –f “/usr/bin/srun –N 4 -n 4 ./smg2000 –n 100 100 100” mpi

The above command will bring up the GUI window with the executable loaded to run the MPI experiment on localhost.  All the user would need to do is push the "Run" button on the GUI and wait for the results to be displayed.  The path to the executable can be a relative path.

Quotes are used around the –f option if the executable has parameters or arguments, such as in the MPI application shown above.

If using the Open|SpeedShop wizard capability to guide you through the process of creating and starting a performance experiment, just call out the openss command with no arguments.  Both the online (dynamic instrumentation) and offline (static linking) modes of operation are supported.

prompt:> openss

Example 3:  Using Interactive CLI

The user wants use the command line interface and knows their executable and the type of experiment they want to run (User Time).

General syntax:
prompt:> openss -cli
expcreate [-i offline | -i online ] -f < pathToExecutable>/executable_name <experiment_type>
expgo
expview

Real example:
prompt:> openss -cli
expcreate -f “/home/jeg/matmul < matmul.input” usertime
expgo
expview


The openss command above will invoke the command line interface in its initial state.   The user would then enter the experiment create command (expcreate) and specify the host, executable file and experiment name.  Quotes are used around the –f option if the executable has parameters as does the application shown above. The command to run the experiment (expgo) will start the user application and performance experiment running.   Currently the Open|SpeedShop command line interface is asynchronous.   The user will get a prompt back immediately after issuing the expgo command.   This allows the user to issue other commands and/or monitor performance data prior to completion of a long running job.   If performance experiment data is available, the performance result viewing command (expview) can be used to view the data on demand.  Open|SpeedShop will return the data that it currently knows about.

Example 4:  Using Batch

The user wants use the batch interface and knows the executable and the experiment they want to run (PC Sampling.   There are two methods for accomplishing this.   The default method would be to use the "openss -batch" command line to specify all the arguments and let Open|SpeedShop run the experiment and display the performance data using the default view.   The command for the example defined above would be:


General syntax:
prompt:> openss -batch -f < pathToExecutable>/executable_name <experiment_type>

Real example:
prompt:> openss -batch -f ./sweep3d.single pcsamp


In the above example, Open|SpeedShop uses the default view to display the performance data.

If the user wants to deviate from the default view or wants to run other Open|SpeedShop CLI commands there is a way to do this.   The user creates a file of Open|SpeedShop CLI commands and pipes them into Open|SpeedShop in this form: "openss -batch < batch.commands".

For example: Create a file (batch.input) containing the command line commands:


expcreate -f ./sweep3d.single pcsamp

expgo
expview

Then execute the Open|SpeedShop batch tool by using the "-batch" argument on the "openss" command:

prompt:> openss -batch < batch.input

The Open|SpeedShop PC Sampling experiment will be run and the output will be output to stout.

Viewing Open|SpeedShop Performance Data post-mortem

As mentioned above, all the various ways of running Open|SpeedShop create a database file that has the suffix “.openss”. This file contains all the symbols, time stamps, and performance data information to view the results of the Open|SpeedShop performance experiment which was run on the user’s application.


Example 1:  Using the GUI

To open the Open|SpeedShop database file with the graphical user interface (GUI), you can use either of two openss command forms:

  • Use the command “openss” and follow the wizard instructions to open the saved database file

  • Use the command “openss –f <database_filename>.openss”. This command will load the database information into the GUI in one step. When the GUI is raised the data will be presented for viewing.


Example 2:  Using the Interactive CLI

To open the Open|SpeedShop database file with the interactive command line user interface (CLI), you can use either of two openss command forms:

  • Use the command “openss -cli” and use the exprestore –f <database_filename>.openss” to open the saved database file

  • Use the command “openss -cli –f <database_filename>.openss”. This command will load the database information into the CLI in one step. When the CLI is opened the data will be loaded for viewing.

In both cases above, use the “expview” command for the basic default view of the data. Use the “help expview” command for more interesting options to filter, format, and view the performance report data.
